Gestion de Stock Système Multi-Sites Laravel

Application web complète de gestion de stock avec architecture Laravel MVC, système d'alertes automatisé, traçabilité complète et gestion multi-sites avec catégories dynamiques

Laravel 10 / PHP 8.2
MySQL / Eloquent ORM
Alertes Automatisées
Stock Manager

Contexte du Projet

Durant mon stage au sein du groupe ACPPA, j'ai développé une application web complète de gestion de stock en Laravel. Cette solution innovante visait à organiser les produits par catégories et lieux, tout en assurant un suivi précis des mouvements et des actions via un système de logs automatisé.

Objectif Principal

Créer une solution robuste de gestion de stock permettant l'inventaire automatisé, le suivi des équipements et la gestion des interventions techniques avec traçabilité complète des actions.

Le projet s'inscrivait dans une démarche d'optimisation des processus internes du groupe ACPPA, nécessitant une solution capable de gérer plusieurs sites simultanément avec des niveaux d'accès différenciés et un système d'alertes intelligent.

Structure MVC Détaillée

Contrôleurs (7)

  • DashBoardController - Interface principale avec alertes automatiques
  • ListeProduitsController - CRUD complet produits avec codes uniques
  • ListeCategoriesController - Gestion catégories par lieu
  • ListeLieuController - Administration multi-sites
  • LogController - Système de logs avec fichiers datés
  • LoginController - Authentification sécurisée
  • Controller - Contrôleur de base Laravel

Modèles Eloquent (5)

  • Produit - Entité principale (nom, quantité, limite, code unique)
  • Categorie - Organisation par catégories liées aux lieux
  • Lieu - Gestion multi-sites avec lieu par défaut
  • Admin - Gestion utilisateurs et authentification
  • CategorieLieu - Relations complexes many-to-many

Vues Blade (8+)

  • dashboardView - Tableau de bord avec alertes
  • ajouterproduitView - Formulaires produits
  • voirstockView - Visualisation stock par lieu
  • ajouterlieuView - Gestion lieux de stockage
  • ajoutercategorieView - Création catégories
  • voircategorieView - Liste catégories
  • logCompleteView - Historique complet
  • admin - Layout principal

Structure Base de Données

Table `produits`

id_produit Primary Key (Auto-increment)
nom_produit String (Nom du produit)
quantite Integer (Quantité en stock)
limite Integer (Seuil d'alerte)
description String Nullable (Description)
code_produit String Unique (Code produit)
categorie_id Foreign Key → categories

Table `categories`

id_categorie Primary Key
nom_categorie String (Nom catégorie)
id_lieu Foreign Key → lieux

Table `lieux`

id_lieu Primary Key
nom_lieu String (Nom du lieu)
default_lieu Boolean (Lieu par défaut)

Table `admin`

id Primary Key
username String (Nom utilisateur)
password String (Mot de passe hashé)

Routes et Endpoints

Dashboard

GET /DashBoardView Interface principale avec alertes

DELETE /deleteproduitDash Suppression produit depuis dashboard

Produits

GET /VoirStock Visualisation stock complet
